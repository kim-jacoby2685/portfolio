Commitment to Accessibility

I believe everyone should be able to access and use the web, regardless of ability. This portfolio was designed and built to demonstrate inclusive design practices and to provide an accessible experience for all visitors.

I continually review and improve this site to address accessibility barriers and to stay current with evolving standards and best practices.

Conformance Goal

The goal of this portfolio is to meet Web Content Accessibility Guidelines (WCAG) 2.2 Level AA standards. WCAG defines requirements for making web content more accessible to people with disabilities across three levels of conformance: Level A, Level AA, and Level AAA.

Level AA is widely recognized as the standard for accessible public-facing websites and is the target for this portfolio.

Accessibility Features

This portfolio includes the following accessibility features:

  • Semantic HTML5 structure with landmark regions (header, navigation, main, footer)
  • One level 1 heading per page with a logical heading hierarchy
  • Skip to main content link with programmatic focus support
  • Scroll padding so focused elements are not hidden behind the sticky header
  • Visible focus indicators that meet WCAG 2.2 Focus Appearance requirements
  • Meaningful alternative text for informative images
  • Sufficient color contrast for text and interactive elements
  • Links distinguishable by underline, not color alone
  • Descriptive link text that identifies the purpose of each link
  • Language attribute set on the HTML element
  • Unique and descriptive page titles
  • Support for reduced motion via prefers-reduced-motion
  • Support for high contrast mode via forced-colors
  • Limited ARIA usage — native HTML elements are preferred

Keyboard Navigation

All functionality on this portfolio is available using a keyboard. You can navigate between links and buttons using Tab and Shift + Tab. Interactive elements can be activated with Enter or Space.

Keyboard navigation support includes:

  • Logical tab order that follows the visual layout
  • Visible focus indicators on all interactive elements
  • Skip to main content link as the first focusable element on every page
  • Mobile navigation menu operable by keyboard, with Escape to close
  • No keyboard traps — focus can always be moved away from any component

Responsive Design

This portfolio uses a responsive layout that adapts to different screen sizes, orientations, and zoom levels. Content reflows without requiring horizontal scrolling at viewport widths as narrow as 320 pixels and supports browser zoom up to 400%.

Responsive design support includes:

  • Flexible layouts using CSS Grid and Flexbox
  • Scalable text and images that resize with the viewport
  • A collapsible navigation menu on smaller screens
  • Touch-friendly target sizes for navigation controls on mobile devices

Screen Reader Compatibility

This portfolio is designed to be compatible with screen readers and other assistive technologies. Semantic HTML provides structure and meaning that assistive technologies can interpret and convey to users.

Screen reader compatibility features include:

  • Landmark regions for quick navigation between page areas
  • Descriptive headings that summarize each section
  • Current page indication in navigation using aria-current="page"
  • Expanded/collapsed state announced on the mobile menu button via aria-expanded

This site has been tested with screen reader navigation patterns using landmark navigation, heading navigation, and link interaction workflows.
